# 💬 mavnezz-os FAQ

Common questions and solutions for managing a mavnezz-os system.

## 🚀 Getting Started

### **❄ What is mavnezz-os?**

mavnezz-os is a personal flake-based NixOS configuration that manages multiple machines (desktop + laptop) from a single repo, built around the Niri scrollable-tiling Wayland compositor with the Noctalia shell.

**Key Features:**
- Multi-host configuration management
- Niri scrollable-tiling Wayland compositor with the Noctalia shell
- Custom `dcli` management tool
- Flake-based reproducible builds

### **🔧 What is dcli?**

`dcli` is a command-line tool that wraps common multi-host management tasks: building configurations, switching hosts, system maintenance.

**Common dcli commands:**
```bash
dcli build <hostname>      # Build configuration for a specific host
dcli deploy <hostname>     # Build and switch to configuration
dcli status                # Show current system status
dcli list-hosts            # List available host configurations
dcli cleanup               # Clean up old system generations
dcli update                # Update flake and rebuild system
```

See `dcli.md` for the full command reference.

## 🏠 Host Management

### **🖥️ How do I add a new computer?**

1. Create a new directory under `devices/<form>/<host>/` with `default.nix` (enabling `workstation.*` options) and `hardware.nix` (generated by `nixos-generate-config`).
2. Add the host to `nixosConfigurations` in `flake.nix`.
3. Test build: `dcli build <hostname>`.

### **⚙️ How do I switch between different host configurations?**

```bash
# Using dcli (recommended)
dcli deploy homework

# Using standard NixOS rebuild
sudo nixos-rebuild switch --flake ~/mavnezz-os#homework
```

### **🔄 How do I update my system?**

```bash
# Update everything with dcli
dcli update

# Or manually
cd ~/mavnezz-os
nix flake update
sudo nixos-rebuild switch --flake .#<hostname>
```

## 🎮 Hardware & Graphics

### **🖥️ How do I configure multiple monitors?**

Edit the relevant Niri KDL config (`config/niri/config.desktop.kdl` or `config.laptop.kdl`) and add/adjust `output` blocks:

```kdl
output "DP-2" {
    mode "2560x1440@144"
    position x=0 y=0
    scale 1.0
}

output "HDMI-A-1" {
    mode "1920x1080@60"
    position x=2560 y=0
    scale 1.0
}
```

Use `niri msg outputs` to discover available output names.

## 🔧 Configuration & Customization

### **🎨 How do I change the wallpaper and theme?**

Wallpapers and shell theming are driven through Noctalia. Drop new wallpapers into `wallpapers/` and pick them via the Noctalia settings UI, or edit `config/niri/noctalia.kdl` directly.

Available wallpapers are in the `wallpapers/` directory.

### **🌍 How do I change my timezone?**

Edit the relevant device module (`devices/<form>/<host>/default.nix`) and set `time.timeZone`.

### **⌨️ How do I change keyboard layout?**

Adjust `services.xserver.xkb` (or the equivalent console settings) in the device module.

### **📦 How do I install additional software?**

1. **Baseline packages (all hosts):** Edit `modules/packages.nix`. Hosts opt in via `workstation.baseline.packages.{tools,dev,apps}`.
2. **Host-specific packages:** Add to `environment.systemPackages` inside `devices/<form>/<host>/default.nix`.

## 🚨 Troubleshooting

### **⚠️ My system won't boot after an update**

1. **Select previous generation at boot**
2. **Or roll back:**
   ```bash
   sudo nixos-rebuild switch --rollback
   ```

### **🔍 How do I diagnose system issues?**

```bash
# Generate diagnostic report
dcli diag

# Check system logs
journalctl -f

# Test configuration without switching
dcli build <hostname>
```

### **💾 How do I free up disk space?**

```bash
# Clean old generations
dcli cleanup

# Manual cleanup
nix-collect-garbage -d
sudo nix-collect-garbage -d
```

## 🏗️ Building & Development

### **🔨 How do I test changes without switching?**

```bash
dcli build <hostname>

# Or manually
nixos-rebuild build --flake ~/mavnezz-os#<hostname>
```

### **📋 How do I back up my configuration?**

The entire configuration lives in `~/mavnezz-os/`. Commit it to git and push.
